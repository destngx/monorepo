**Description**:
Implement a Go-based CLI for the GitHub Device Flow. This is the **AI Authentication Client** to get a GitHub Copilot bearer token for use in the Go engine.

- **Prerequisites**: Lives in `apps/wealth-management-engine/cmd/github_copilot_setup`.
- **Logic**: Use GitHub Device Choice Flow (RFC 8628).
  **Acceptance Criteria**:
- Requests device and user code from GitHub `login/device/code`.
- Displays the user code and verification URI in the terminal.
- Polls for the access token from `login/oauth/access_token`.
- Saves the `GITHUB_TOKEN` to `.env.local` upon successful authorization.

**Implementation Notes**:
- Strictly uses standard libraries (`net/http`, `encoding/json`) for the flow.
- Includes waiting and status polling with interval-based retries.
- Verified with standard `dev` credentials from GitHub Copilot.

**Development Commands**:
```bash
# Run the setup script
bunx nx run wealth-management-engine:run --args="cmd/github_copilot_setup/main.go"
```
