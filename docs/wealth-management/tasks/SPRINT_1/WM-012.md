**Description**:
Provide first-class API discoverability and contract documentation by exposing an OpenAPI spec and Swagger UI directly from the Go engine.

**Acceptance Criteria**:

- ✅ `GET /api/openapi.json` serves a valid OpenAPI spec generated dynamically from registered Fiber routes.
- ✅ `GET /api/docs` serves Swagger UI and renders `/api/openapi.json`.
- ✅ Swagger tags are grouped by API domain segment (for example, `/api/market/*` routes are tagged `market`).
- ✅ Spec excludes documentation infrastructure endpoints (`/api/docs`, `/api/openapi.json`) from `paths`.
- ✅ Spec includes current market capability endpoints and real AI endpoints (`/api/ai/*`) discovered from code routes.
- ✅ Automated tests verify spec and docs endpoints.

**Implementation Notes**:

- Added code-generated OpenAPI handler:
  - `apps/wealth-management-engine/adapter/fiber/openapi_handler.go`
  - Endpoints:
    - `GET /api/openapi.json`
    - `GET /api/docs`
  - Behavior:
    - Derives operations from Fiber route registry at runtime
    - Derives tags from second path segment under `/api/`
    - Excludes `/api/docs` and `/api/openapi.json` from OpenAPI `paths`
- Wired routes in app bootstrap:
  - `apps/wealth-management-engine/main.go`
- Added BDD-style endpoint tests:
  - `apps/wealth-management-engine/adapter/fiber/openapi_handler_test.go`
