**Description**:
Implement the prompt management engine in Go for complex task orchestration.

- **Logic**: Support for System/User/Assistant role-tagging.
- **Logic**: JSON Response enforcing (e.g., "Respond only in valid JSON format").
- **Custom Prompts**: Template-based injection for specialized tasks (e.g., Investment Briefing).
  **Acceptance Criteria**:
- Endpoint `/api/ai/json` returns a perfectly parsed Go struct from an AI JSON response.
- Response contract guarantees non-empty structured fields (`persona`, `summary`, `actions`, `roles`) even when provider output is partially empty.
- Successfully demonstrates a three-role interaction (System Persona + User Query + AI History).

**Implementation Notes**:

- **Endpoint**:
  - Added `POST /api/ai/json` in `adapter/fiber/ai_handler.go`.
- **Route Policy**:
  - Structured JSON generation is exposed under real AI API namespace `/api/ai/*` (not `/api/test/*`).
- **Role Synthesis**:
  - Added explicit 3-role message contract in `domain/structured_prompt.go`:
    - `system`
    - `assistant` (history)
    - `user`
- **JSON Enforcement + Parsing**:
  - Added provider-agnostic port method `CompleteJSON(...)` in `port/ai.go`.
  - Copilot adapter now supports non-stream JSON completion with `response_format: { "type": "json_object" }`.
  - Service method `GenerateStructuredJSON(...)` parses AI JSON directly into Go struct `StructuredAIResponse`.
  - Service normalizes empty structured fields to safe defaults so `/api/ai/json` contract remains stable in e2e.
- **TDD/BDD Tests**:
  - Added BDD-style test in `service/ai_service_test.go` validating:
    - Three-role interaction order (System → Assistant → User)
    - Default model fallback (`gpt-4.1`)
    - Successful parsed Go struct output from AI JSON
    - Fallback normalization for empty structured output fields
