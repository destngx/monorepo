**Description**:
Implement a **capability-based market provider architecture** that routes data requests by domain capability (e.g., `GetTicker`, `GetExchangeRate`, `GetPriceSeries`) rather than by provider actions. Fmarket and Vnstock are **peer providers** in a unified system where routing decisions are **config-driven** per capability.

**Key Design Principle**: The `port.MarketProvider` interface is **stable** regardless of how many providers exist. Routing logic is **injected** into the service layer via configuration, enabling multi-provider extensibility without code changes.

- **Provider Interface**: Extend `port.MarketProvider` with capability methods (generic, not action-based).
- **Fmarket Implementation**: Implement all capabilities that Fmarket.vn supports (IFC tickers, equity tickers, exchange rates, bank rates, price series for gold and other assets).
- **Capability Routing**: Service layer routes by capability type (e.g., `equity`, `ifc`, `USD_VND`, `gold_sjc`) with provider priority chains from injected config.
- **Price Series Model**: Gold pricing is modeled as a **time-series** (`GetPriceSeries`) with provider-specific schemas (Fmarket: SJC bid/ask + USD world price; future providers may differ).
- **Schema Normalization**: Service layer normalizes provider-specific schemas to unified domain models where applicable; returns provider-native schema for heterogeneous data (gold).
- **Fallback Chains**: Each capability maps to a provider chain (e.g., IFC tickers: [Fmarket, Vnstock]; equity tickers: [Vnstock, Fmarket]; gold: [Fmarket]).
- **Caching**: Capability-level cache keys enable unified caching across multiple providers.

**Implementation Notes**:

- **Port Interface Design**:
  - `port/market_provider.go` defines stable, capability-based interface (independent of provider count)
  - Methods accept generic type parameters (e.g., `tickerType TickerType`, `from, to Currency`)
  - All methods include context for cancellation and timeout support
  - Each method is a **capability**, not an action

  ```go
  type MarketProvider interface {
    Name() string
    Health(ctx context.Context) error
    GetTicker(ctx context.Context, symbol string, tickerType TickerType) (*Ticker, error)
    GetExchangeRate(ctx context.Context, from, to Currency) (*ExchangeRate, error)
    GetPriceSeries(ctx context.Context, symbol string, seriesType SeriesType) (interface{}, error)
    GetBankInterestRate(ctx context.Context) ([]*BankRate, error)
  }

  // SeriesType for time-series data
  type SeriesType string
  const (
    SeriesTypeEquityOHLC SeriesType = "equity"        // OHLC data
    SeriesTypeIFCNav     SeriesType = "ifc"           // IFC NAV history
    SeriesTypeGoldSJC    SeriesType = "gold_sjc"      // Gold SJC bid/ask (VND)
    SeriesTypeGoldUSD    SeriesType = "gold_usd"      // Gold world price (USD)
    SeriesTypeForex      SeriesType = "forex"         // FX rate history
  )
  ```

- **Fmarket Provider Implementation**:
  - `adapter/market/fmarket/provider.go` implements all 4 capabilities
  - Maps Fmarket API actions to capabilities:
    - `GetTicker(symbol, "ifc")` → `getProductDetails` API (IFC details)
    - `GetTicker(symbol, "gold")` → `getGoldProducts` API (SJC physical gold bid/ask history, VND)
    - `GetExchangeRate("USD", "VND")` → `getUsdRateHistory` API
    - `GetPriceSeries(symbol, "gold_usd")` → `getGoldProducts` API (world gold price history, USD)
    - `GetBankInterestRate()` → `getBankInterestRates` API
  - Returns error for unsupported types (e.g., `GetTicker(symbol, "equity")` → fallback to Vnstock)
  - **Schema note**: `GetTicker("gold")` returns **normalized `Ticker` struct** (unified format). `GetPriceSeries("gold_usd")` returns **provider-native schema** because world price series has different semantics than SJC bid/ask
- **Vnstock Provider Refactoring**:
  - Update `adapter/market/vnstock/provider.go` to implement same interface
  - Maps Vnstock APIs to same capability methods:
    - `GetTicker(symbol, "equity")` → Vnstock ticker API
    - `GetExchangeRate()` → Vnstock FX rates (fallback for USD/VND)
    - `GetPriceSeries(symbol, "equity")` → OHLC history (if available)
  - Returns error for unsupported capabilities (e.g., `GetTicker(symbol, "gold")`, `GetBankInterestRate()`)

- **Schema Normalization Strategy**:
  - **Homogeneous ticker data** (equity, ifc, gold): Service layer normalizes to unified `Ticker` struct
    - Different provider schemas (bid/ask, spot, high/low/close) → normalized `Ticker` struct
    - Client receives consistent schema regardless of provider source
  - **Heterogeneous time-series** (price history): Service returns **provider-native schema** + metadata
    - Fmarket gold_usd: `[{price, timestamp}, ...]` (world gold price in USD)
    - Binance crypto (future): `[{open, high, low, close, volume}, ...]` (OHLC)
    - **Rationale**: Different time-series have fundamentally different semantics; forcing normalization creates abstraction leaks
    - **Client responsibility**: Interpret schema based on `seriesType` and `provider` metadata

  ```json
  // Example: Gold SJC ticker (normalized)
  {
    "symbol": "SJC",
    "type": "gold",
    "provider": "fmarket",
    "bid": 82500000,
    "ask": 82700000,
    "currency": "VND",
    "timestamp": "2026-03-30T14:30:00Z"
  }

  // Example: Gold USD price series (provider-native schema)
  {
    "seriesType": "gold_usd",
    "provider": "fmarket",
    "currency": "USD",
    "data": [
      {"price": 2150.50, "timestamp": "2026-03-30T14:30:00Z"},
      {"price": 2151.25, "timestamp": "2026-03-30T15:00:00Z"}
    ]
  }

  // Example: Equity OHLC (normalized)
  {
    "symbol": "ACB",
    "type": "equity",
    "provider": "vnstock",
    "open": 31500,
    "high": 32000,
    "low": 31200,
    "close": 31800,
    "timestamp": "2026-03-30"
  }
  ```

- **Capability Routing Config** (Injected):
  - Config defines provider priority per capability+type (YAML/JSON or ENV-driven)
  - Example:
    ```yaml
    capabilities:
      getTicker:
        equity: [vnstock, fmarket] # Stock tickers: Vnstock primary
        ifc: [fmarket, vnstock] # Fund certificates: Fmarket primary
        gold: [fmarket] # Gold SJC bid/ask: Fmarket only
      getExchangeRate:
        USD_VND: [fmarket, vnstock] # VN rates: Fmarket primary
      getPriceSeries:
        equity: [vnstock, fmarket] # Equity OHLC history
        gold_usd: [fmarket] # World gold price (USD): Fmarket only
      getBankInterestRate: [fmarket] # Bank rates: Fmarket only
    ```
  - Service layer loads this config at startup
  - Allows A/B testing and runtime reconfiguration without code changes
  - **Gold note**: `getTicker.gold` returns SJC bid/ask (normalized Ticker). `getPriceSeries.gold_usd` returns world price history (provider-native schema)
- **Service Layer Routing**:
  - `service/market_provider_service.go` orchestrates capability dispatch
  - For each capability request:
    1. Look up provider chain from routing config
    2. Try each provider in priority order
    3. Return first successful response
    4. Log fallback to secondary provider for monitoring

  ```go
  func (s *Service) GetTicker(ctx, symbol, tickerType) (*Ticker, error) {
    chain := s.routing.ChainFor("getTicker", tickerType)
    for _, providerName := range chain {
      provider := s.providers[providerName]
      ticker, err := provider.GetTicker(ctx, symbol, tickerType)
      if err == nil { return ticker, nil }
      s.log.Warn("provider failed, trying fallback", providerName)
    }
    return nil, ErrAllProvidersFailed
  }

  func (s *Service) GetPriceSeries(ctx, symbol string, seriesType SeriesType) (interface{}, error) {
    chain := s.routing.ChainFor("getPriceSeries", string(seriesType))
    for _, providerName := range chain {
      provider := s.providers[providerName]
      data, err := provider.GetPriceSeries(ctx, symbol, seriesType)
      if err == nil { return data, nil } // Return provider-native schema with metadata
      s.log.Warn("provider failed, trying fallback", providerName)
    }
    return nil, ErrAllProvidersFailed
  }
  ```

- **Caching Strategy**:
  - Capability-level cache keys: `market:{capability}:{type}:{symbol}` or `market:{capability}:{params_hash}`
  - TTL sourced from domain constant `MARKET_DATA_TTL` (default: 300-600s, configurable per capability)
  - Cache enabled at service layer before provider dispatch
  - Force-refresh via `skipCache` parameter or explicit cache invalidation
- **Header Injection** (Fmarket-specific):
  - Fmarket provider applies institutional headers to all outbound requests:
    - `User-Agent`: High-fidelity browser string (macOS/Firefox class)
    - `Origin`: `https://fmarket.vn` (CSRF bypass)
    - `Referer`: `https://fmarket.vn/` (referer-policy bypass)
    - `F-Language`: `vi` (Vietnamese locale)
  - Headers embedded in provider, not in handler
- **HTTP Handler Layer**:
  - `adapter/fiber/market_handler.go` exposes generic capability endpoints (not action-based):
    - `GET /api/market/ticker?symbol=ACB&type=equity` → routes to `GetTicker(ACB, equity)`
    - `GET /api/market/ticker?symbol=FUND001&type=ifc` → routes to `GetTicker(FUND001, ifc)`
    - `GET /api/market/ticker?symbol=SJC&type=gold` → routes to `GetTicker(SJC, gold)` - returns gold bid/ask
    - `GET /api/market/exchange-rate?from=USD&to=VND` → routes to `GetExchangeRate(USD, VND)`
    - `GET /api/market/price-series?symbol=GOLD&type=gold_usd` → routes to `GetPriceSeries(GOLD, gold_usd)` - returns world gold price history
    - `GET /api/market/bank-rates` → routes to `GetBankInterestRate()`
  - Ticker responses include normalized schema (uniform across providers)
  - PriceSeries responses include metadata: `provider`, `seriesType`, `currency` so client interprets provider-native schemas
  - Legacy action-based endpoint `POST /api/fmarket` can be implemented as adapter layer (maps actions to capabilities for backward compatibility)
- **Dependency Injection** (main.go):
  - Providers registered as singletons
  - Routing config loaded from env/file
  - Service instantiated with providers + routing config:
    ```go
    providers := map[string]port.MarketProvider{
        "fmarket": NewFmarketProvider(httpClient, config),
        "vnstock": NewVnstockProvider(httpClient, config),
    }
    routing := LoadRoutingConfig("config/routing.yml")
    service := NewMarketProviderService(providers, routing, cache)
    ```
- **Error Handling**:
  - Provider returns `nil, err` if it doesn't support the requested capability
  - Service logs fallback attempt and tries next provider in chain
  - If all providers fail, return comprehensive error with tried providers list
  - Empty results (e.g., no funds) return 200 with empty slice (valid state)
  - Network timeout: 10s per provider; service aggregates timeouts
- **TDD/BDD Tests**:
  - Unit tests per provider: verify capability implementations and API mapping
  - Integration tests: verify routing config is respected
  - Tests validate: (1) correct provider chosen for capability+type, (2) fallback triggered on failure, (3) cache properly keyed by capability
  - Mock providers for deterministic testing
  - Mock routing config for test scenarios
- **Configuration**:
  - Routing config: `config/routing.yml` (loaded at startup, injectable from env)
  - Per-provider timeout: `10s` (configurable)
  - Cache TTL: `300-600s` per capability (configurable in routing config)
  - Base URLs: `FMARKET_BASE_URL`, `VNSTOCK_BASE_URL` from env
