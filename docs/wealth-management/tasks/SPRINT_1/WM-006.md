**Description**:
Implement the initial AI connection client in the Go backend using the GitHub Copilot subscription.

- **Logic**: Use the `api.githubcopilot.com` endpoint with the Copilot Bearer token.
- **Model Default**: Configure for "GPT-4.1" (Latest available LLM).
- **Goal**: Confirm the backend can send a system prompt and receive a valid JSON/Text response.
  **Acceptance Criteria**:
- CLI tool or endpoint `/api/test/ai/stream` returns a successful real-time completion.
- Handles the custom "Copilot Token Exchange" flow natively in Go.

**Implementation Notes**:

- **Token Exchange**: Native Go implementation calls GitHub Copilot token endpoint:
  - `GET {GITHUB_API_BASE_URL}/copilot_internal/v2/token`
  - Uses header `Authorization: token {GITHUB_TOKEN}`
  - Caches bearer token in-memory with expiry buffer.
- **Copilot Streaming Call**:
  - `POST {COPILOT_API_BASE_URL}/chat/completions` with `stream: true`.
  - Default model set to `gpt-4.1` (`COPILOT_DEFAULT_MODEL` override supported).
  - Includes Copilot-compatible headers (`Editor-Version`, `Editor-Plugin-Version`).
- **Hexagonal Wiring**:
  - Adapter: `adapter/ai/copilot/client.go`
  - Port: `port/ai.go`
  - Service: `service/ai_service.go`
  - Fiber endpoint: `POST /api/test/ai/stream` in `adapter/fiber/ai_handler.go`
  - Configuration adapter: `adapter/config/config.go`
- **Env Configuration**:
  - `GITHUB_TOKEN` (required)
  - `GITHUB_API_BASE_URL` (optional, default `https://api.github.com`)
  - `COPILOT_API_BASE_URL` (optional, default `https://api.githubcopilot.com`)
  - `COPILOT_DEFAULT_MODEL` (optional, default `gpt-4.1`)
- **TDD/BDD Tests**:
  - Adapter tests validate token exchange, request headers, stream mode, and token-cache behavior.
  - Provider-level adapter tests validate:
    - Copilot streaming path (`StreamCompletion`)
    - Copilot structured JSON path (`CompleteJSON`)
    - token exchange + cache behavior
  - Service test validates default model selection when request model is empty.
