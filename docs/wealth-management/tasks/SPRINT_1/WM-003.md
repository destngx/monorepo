**Description**:
Port the legacy TypeScript OAuth client logic into Go. This is the **Core DB Client** for the entire platform.

- **Prerequisites**: Must reuse existing `GOOGLE_SHEETS_ID` and OAuth secrets from `.env`.
- **Logic**: Use `golang.org/x/oauth2` and `google.golang.org/api/sheets/v4`.
  **Acceptance Criteria**:
- Go service can read the `Accounts` sheet tab successfully.
- Implements the "Refreshed Bearer Token" logic natively in Go.
- Integration tests for `readSheet` and `appendRow`.

**Implementation Notes**:

- **Env Loading**: Added root/app `.env.local` loading for runtime configuration without hardcoding credentials.
- **Hexagonal Wiring**:
  - `adapter/google/` contains the Google Sheets OAuth + API client.
  - `domain/accounts.go` defines the `Accounts` domain payload for the `Accounts` tab.
  - `service/sheets_service.go` exposes the application-facing use case for `Accounts`.
  - `adapter/fiber/sheets_handler.go` exposes `GET /api/sheets/accounts` for runtime verification.
- **OAuth Flow**: Uses native Go `oauth2` refresh-token exchange to obtain bearer tokens on demand before Sheets API calls.
- **API Behavior**:
  - `ReadSheet` requests `UNFORMATTED_VALUE` and normalizes cells to `string`.
  - `AppendRow` uses `USER_ENTERED` writes to preserve Sheets-side formatting/formulas behavior.
- **Testing**:
  - Integration tests cover `readSheet` and `appendRow` against a mocked HTTP server.
  - Verified with `go test . ./adapter/... ./domain ./port ./service` using repo-local Go caches.
